# Project Governance and Decision Making

This document outlines the decision-making process and governance model for the GAMA (Global Agri-Mind AI) project.

## 1. Project Role Definitions

* **Users**: Anyone who uses the GAMA application or integrates with GAMA API endpoints.
* **Contributors**: Anyone who contributes code, documentation, designs, or issues to the repository.
* **Maintainers**: Core community members with write access to the main repository who review and merge contributions.
* **Project Leads / Admins**: Representatives responsible for the administrative management of the organization, Open Collective funding accounts, and repository policies.

## 2. Decision-Making Process

We strive to operate through a **consensus-seeking model**:

* **Technical Changes**: Discussed via GitHub issues and Pull Request comments. Maintainers review changes to ensure quality and compatibility. Approval from at least one core maintainer is required to merge code.
* **Major Architecture & Policy Decisions**: Discussed publicly on GitHub issues. If a consensus cannot be reached naturally, the Project Leads will make the final decision.
* **Financial Governance (Open Collective)**: All financial expenses (hosting costs, domain names, promotional assets, contributor bounties) must be approved by the Project Leads and documented transparently on our Open Collective page.
